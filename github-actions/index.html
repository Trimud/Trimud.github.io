Github Actions

GitHub Actions are a continuous integration tool which offers developers a new way to automate workflows for their web projects. Each action can be combined with other actions to create a workflow.

Why Github Actions?

  • Easy setup
  • SaaS
  • Multiple events available
    (deployment, scheduled, push (filtered by branch and file type), pull request created, etc.)
  • Integration with OSF JIRA Webhooks Tool (Presentation)
  • Can send notification to Slack channel

Job steps

  • Multiple action steps are supported.
  • They can run in parallel or be dependant on each other.
  • Data can be used from one step to another using Github Actions Artifacts
  • Step can be run conditionally
Deployment event
name: Deploy
on: ['deployment']

Download Build Suite

build-suite:
  name: Download Build Suite
  runs-on: ubuntu-latest

  steps:
    - name: Checkout
      uses: actions/checkout@v2
      with:
        repository: OSFGlobal/build-suite-sfcc
        token: ${{secrets.GH_TOKEN}}

    - name: Upload build-suite artifact
      uses: actions/upload-artifact@v1
      with:
        name: build-suite-sfcc
        path: ./

  needs: [build-suite] // Download build suite artifact from previous step
  runs-on: ubuntu-latest

  steps:

      uses: "actions/setup-node@v1"
      with:

      run: |
        npm i
        npm run compile:dev

    - name: Download build-suite artifact
      uses: actions/download-artifact@v1
      with:
        name: build-suite-sfcc

    - name: Install Build Suite and deploy to ${{github.event.deployment.environment}}
      if: github.event.deployment.payload.site_import == false
      env:
        SITE_IMPORT: ${{github.event.deployment.payload.site_import}}
        TWO_FACTOR: ${{github.event.deployment.payload.two_factor}}
        BUILD_NUMBER: ${{github.event.deployment.payload.build_number}}
        HOST: ${{github.event.deployment.payload.instance}}
        SF_USERNAME: ${{secrets.SF_USERNAME}}
        SF_PASSWORD: ${{secrets.SF_PASSWORD}}
        CLIENT_ID: ${{secrets.CLIENT_ID}}
        CLIENT_SECRET: ${{secrets.CLIENT_SECRET}}
        CERT_PASSWORD: ${{secrets.CERT_PASSWORD}}
      run: |
        cd build-suite-sfcc
        npm i
        grunt dist --project=projects/vam-sfcc/config --target=deploy

    - name: Install Build Suite, deploy to ${{github.event.deployment.environment}} and import site data
      if: github.event.deployment.payload.site_import == true
      env:
        SITE_IMPORT: ${{github.event.deployment.payload.site_import}}
        TWO_FACTOR: ${{github.event.deployment.payload.two_factor}}
        BUILD_NUMBER: ${{github.event.deployment.payload.build_number}}
        HOST: ${{github.event.deployment.payload.instance}}
        SF_USERNAME: ${{secrets.SF_USERNAME}}
        SF_PASSWORD: ${{secrets.SF_PASSWORD}}
        CLIENT_ID: ${{secrets.CLIENT_ID}}
        CLIENT_SECRET: ${{secrets.CLIENT_SECRET}}
        CERT_PASSWORD: ${{secrets.CERT_PASSWORD}}
      run: |
        cd build-suite-sfcc
        npm i
        grunt dist --project=projects/vam-sfcc/config --target=deploy
        grunt initSite --project=projects/vam-sfcc/config --target=deploy

    - name: Notify slack channel
      uses: 8398a7/action-slack@v2
      with:
        status: ${{ job.status }}
        author_name: Github Actions
      env:
        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}
      if: always()

Use Postman to deploy

Build Request
POST https://api.github.com/repos/OSFGlobal/<PROJECT>/deployments
Headers
Authorization: token {{GITHUB_TOKEN}}
Accept: application/vnd.github.ant-man-preview+json
Content-Type: application/json
Body
{
	"ref": "develop", // branch or tag name
	"environment": "development", // triggers different action step
	"auto_merge": false, // we don't want default repo branch to be merged automatically
	"payload": {
		"build_number": "0.0.4", // Code version
		"instance": "development-<realm>-<client>.demandware.net",
		"site_import": true, // Should site import be performed?
		"two_factor": false // 2FA for deployments to Staging
	},
	"required_contexts": [] // Bypass checking status context verification against commit status checks
}

Available actions

  • Scheduled/On-Demand deployment to SF instance
  • Scheduled Job Run on Sandbox instances
  • Lint - lints code on each push to feature/bugfix/hotfix branches when any of these type of files is changed: JS/SCSS/ISML/XML
  • Code Review Assistant - lints code on each push to feature/bugfix/hotfix branches when any JS file is changed.
